  • What is new in Wolfpack 13.x.x?
    There are many new things in Wolfpack 13.x.x

  • What are the system hardware requirements?
    While this remains currently untested, here are some recommendations.

    Minimum:
    • 128 MB of RAM.
    • A processor of at least 400mhz.
    • At least 250 MB of free hard drive space.

    Recommended:
    • 256 MB of RAM or greater.
    • A processor of at least 800mhz.
    • At least 500 MB of free hard drive space.

    Naturally, the better your hardware the better your server's performance will be.

  • How do I compile Wolfpack under Linux?
    First you need the latest QT-3.3.x release for your linux distribution.
    You will also need the dev package for this version of QT.
    Note: Most distributions use libqt3-mt and libqt3-mt-dev for package names.

    Distribution Notes

    Debian:
    • Recommended that you use Debian Unstable for latest packages.
    • QT3 Packages: libqt3c102-mt libqt3-mt-dev libqt3-headers qt3-dev-tools
    • Python Packages: python2.3 python2.3-dev
    • Recommended Packages: python2.3-mysqldb python2.3-sqlite python2.3-xml libsqlite0 libsqlite0-dev sqlite sqlite-doc
    • MySQL Packages: libmysqlclient-dev libmysqlclient12 mysql-client mysql-common mysql-server

    Compiling Instructions:
    tar xvfj wolfpack.tar.bz2 -C ./
    cd wolfpack_source_dir/
    chmod +x configure.py
    ./configure.py --help
    # Please read the options and use those that apply to you.
    # Running configure.py should auto detect most options.
    ./configure.py --enable-debug --enable-mysql
    # At this point you can edit Makefile and add optional CFLAGS
    # I personally add -march=athlon -03
    make
    # Grab a snack while it compiles.

  • Can I use the world files for previous Wolfpack versions?
    The simple answer for this is no.
    • Wolfpack 13.x.x no longer uses or supports the old WSC save system. This has been replaced with a database save system.
    • Wolfpack 13.x.x will use spawn scripts for filling in NPCs and there are commands available for adding missing world objects.

    However, if you do wish to try and use it there is an import command which supports WSC/TXT/Sphere save formats. Please note this will NOT import NPCs!
    It is recommended that you look into using the decoration command as an alternative to using import.

  • Help! I get unicode errors with my python scripts!
    This problem should only occur with versions prior to Wolfpack 12.9.8.

    This problem is mostly common with Linux/BSD. Python-2.3 on many distributions comes compiled using UCS4 for Unicode. Wolfpack currently requires that Python-2.3 be compiled with UCS2 for Unicode. If this is the case, you will need to download and manually compile Python-2.3 and add the `--enable-unicode=ucs2 --enable-shared` to the ./configure arguments. After this you can install this Python-2.3 to your /usr/local/ directory or anywhere else. Please be sure to put this new python path in the Wolfpack configure.py arguments!

    Python Fix / Compiling Instructions:
    # Compiling/Installing Python-2.3.x tar xvjf Python-2.3.4.tar.bz2 -C ./
    cd python_source_dir/
    # Please note the --enable-shared and --enable-unicode=ucs2
    ./configure --prefix=/usr/local --enable-unicode=ucs2 --enable-shared
    make
    su root
    make install
    exit

    At this point you need to recompile Wolfpack.
    For information reguarding compiling, please check out:
    • Linux Compile Guide
    • BSD Compile Guide

  • How do I connect to Wolfpack?
    The easiest way is to make sure your server settings and client settings are correctly configured.
    Please check your login.cfg in your Ultima Online installation directory so that it resembles this.
    ;Loginservers for Ultima Online
    ;Do not edit this file or patching will fail! Always save a backup.
    ;LoginServer=login.owo.com,7775
    ;LoginServer=login.owo.com,7775
    ;LoginServer=login.owo.com,7776
    ;LoginServer=login.owo.com,7776
    ;PLEASE NOTE THAT 192.168.1.100 IS AN EXAMPLE OF YOUR IP!
    LoginServer=192.168.1.100,2593
    Now, after you've checked your Client's login configuration, please check your wolfpack.xml file in your Wolfpack director so that it resembles this.
      <group name="LoginServer" >
        <option key="Shard 1" value="My World=192.168.1.100,2592" />
      </group>
      <group name="Network" >
        <option key="Allow Stat Request" value="true" />
        <option key="Allow Unencrypted Clients" value="true" />
        <option key="Enable Gameserver" value="true" />
        <option key="Enable Loginserver" value="true" />
        <option key="Gameserver Port" value="2592" />
        <option key="Loginserver Port" value="2593" />
      </group>
    We support the encryption for the latest clients, so using programs like UOGateway, Ignition, UORice, etc. is completely optional and not required.

  • Help! I can't login?!
    First of all, please check out "How do I connect to Wolfpack?" to see if that resolves your problem.

    If not, then this problem usually occurs when players try to connect to Wolfpack using an outdated client. Due to many packet changes over the years, it has become hard to continue supporting clients below the 4.0.0 value. The Age of Shadows Client is the one we continue to support, and the UO:R client may also work as well. So, if you have this problem, please try running uopatch.exe that is in your Ultima Online Installation to upgrade.
    If this fails, you can try finding the free Age of Shadows Demo Using Google.

  • What ports should I have open?
    By default, Wolfpack uses TCP ports 2592 and 2593.
    Optionally you can enable TCP port 2594 which is for the remote admin page.

  • How can I route to my server behind a router?
    Please Note the following:
    2592 == Default Game Server Port
    2593 == Default Login Server Port
    192.168.1.101 == Your Server's Internal Network IP

    To route to a server that's on an internal network, you need to have TCP ports 2592 and 2593 forwarded to the machines internal IP address.
    For example, say your server has the Internal address of 192.168.1.101.
    Login to your router (most home bought routers can be logged into) and find your forwarding/NAT configuration. Once you do that, you want to configure so that:
    TCP PORT 2592 TO 192.168.1.101
    TCP PORT 2593 TO 192.168.1.101

    For a *nix router using IPTables, you can use something like this:
    iptables -t nat -I PREROUTING -p tcp --dport 2592:2593 -j DNAT --to 192.168.1.101

  • Why is _______ feature is not working the same as on Sphere?
    Wolfpack is NOT Sphere.

    Wolfpack's scripts are designed to function similar to how an official UO shard would function.

    If you want the scripts to function similar to those of Sphere, feel free to modify your shard's scripts to make them do so.
    Please do not ask the developers or file bug reports about this!

  • I'm using the CVS scripts with a release binary, why do I get errors?
    This is a common problem we see. People need to understand that CVS scripts are meant to be used with a CVS binary. Using the bleeding edge scripts with a standard Wolfpack release will result in errors. This is because with script changes there are usually core changes to follow with them.

    Please do not use CVS scripts with a release binary. CVS is for developers or for experienced users and scripters.
